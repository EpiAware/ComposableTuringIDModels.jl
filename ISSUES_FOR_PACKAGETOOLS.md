# EpiAwarePackageTools template gaps

Template/scaffold gaps found while adopting
[EpiAwarePackageTools.jl](https://github.com/EpiAware/EpiAwarePackageTools.jl) in
this package have all been filed upstream. This file is now just a pointer to
those issues and a note of what we carry locally until each is fixed.

| Upstream issue | What it is | Status here |
|---|---|---|

Our Apache-2.0 `LICENSE`/`NOTICE` survive a template sync. |
| [#16](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/16) | JET runner could not analyse a DynamicPPL `@model` package (false `UndefVarErrorReport`s for every `~`-assigned local) | **Fixed upstream & adopted.** The managed `test/jet/runtests.jl` now reads a package-owned `test/jet/jet_config.jl`; ours sets `JET_REPORT_FILTER = dynamicppl_model_filter` (shipped by the kit). The previous local runner override is gone. |
| [#33](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/33) | `scaffold` set up plain Documenter, not DocumenterVitepress (the org docs standard) | **Fixed upstream.** This package already migrated its docs to DocumenterVitepress, so no further change was needed here. |
| [#17](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/17) | `test_explicit_imports` forwards `ignore` only to `check_all_explicit_imports_are_public`, not `check_no_implicit_imports`, so `@reexport` cannot pass | **Open upstream — tracking.** A real kit bug we keep watching. | [#69](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/69) | Scaffolded `docs/make.jl` does not set `deploy_url`, so DocumenterVitepress defaults the VitePress `base` to the `github.io` project subpath (`/EpiAwarePrototype.jl/`). | [#73](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/73) | `update()` re-emits the managed `test.yaml` caller verbatim and does not preserve a package-owned `with:` block, so a deliberate Julia floor/matrix override (here `julia_versions: '["1.11","1","pre"]'` + downgrade `julia_version: '1.11'`) is silently stripped on sync, reverting the endorsed 1.11 floor and breaking the `test (lts=1.10)` + downgrade legs. | **Open upstream — filed during the resync.** A real kit over-reach: until it preserves caller `with:` inputs (or exposes the floor as a scaffold input), a full `update()` cannot be applied without re-adding these overrides by hand. The 1.11 floor is endorsed/intentional here, so the managed `test.yaml` change from `update()` was deliberately **not** taken in this resync. |

## Resync scope note (managed-infra sync, 2026-06-30)

Resynced to the latest `EpiAwarePackageTools` main (v0.1.1) via `update(repo; reviewer = "seabbs", docs_subdomain = true)`. The docs-preview deploy is **already wired** on `main` (the `document.yaml` caller triggers on `pull_request` + the reusable `documentation.yml` builds/comments the preview, `docs/make.jl` sets `push_preview = true`, and `docpreviewcleanup.yaml` cleans it up on close), and the kit no longer bumps reusable-workflow refs (Dependabot owns those via `_preserve_reusable_refs`). So the resync was kept **minimal**: only the non-breaking, in-scope managed-CI deltas were taken — the `# MANAGED by …` header comments on the shared callers and the two new lean org-standard workflows (`cancel-on-close.yaml`, `try-this-pr.yaml`) plus `.gitattributes`.

The remainder of `update()`'s output was **not** taken because, for this deliberately lean prototype (single `ad.yaml` matrix, bespoke DocumenterVitepress `docs/make.jl`, no version/benchmark automation, no Claude bots), it over-reaches — each piece is already filed upstream and tracked here:

- managed `docs/make.jl` `build_docs` rewrite (would clobber the bespoke DocumenterVitepress build and needs a missing package-owned `docs/docs_config.jl`) → [#52](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/52) item 1;
- README badge injection (wrong `License: MIT` badge + per-backend AD badges pointing at non-existent `ad-*.yaml` workflows + a duplicate block) → [#49](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/49), [#52](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/52) item 3;
- `codecov.yml` `after_n_builds: 7` + `fail_ci_if_error: true` → tracked via the per-backend AD assumption ([#49](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/49)) and the closed-as-intended [#34](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/34) (no `CODECOV_TOKEN` here yet);
- `.gitignore` / `Taskfile.yml` drift coupled to the unadopted `build_docs`/per-backend-AD layout → [#65](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/65), [#52](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/52) item 4;
- `CODEOWNERS` / dependabot reviewer+assignee changes → [#72](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/72), [#52](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/52) item 6;
- `test.yaml` Julia-floor `with:` strip → [#73](https://github.com/EpiAware/EpiAwarePackageTools.jl/issues/73) (new).

When #17, #18, #49, #52, #65, #72 or #73 is resolved upstream, adopt the fix and update the rows above.
